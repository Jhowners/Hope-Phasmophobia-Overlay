**停止與記憶力搏鬥。開始與鬼魂搏鬥。**

《恐鬼症 (Phasmophobia)》是一個考驗認知力的遊戲。當一隻怨靈開始獵殺時，您的大腦會切換為「戰鬥或逃跑」模式。在電光火石的瞬間，試圖心算腳步速度或記憶聖木冷卻時間會增加**認知負荷**——這是一種導致猶豫和錯誤的心理鬥爭。

**Hope 恐鬼症疊加層**是您的「第二大腦」。它直接浮動在遊戲上方，減少數據處理工作，讓您能夠將 100% 的注意力放在生存上。

---

## 🧠 「大腦」科學
每次您使用 Alt+Tab 切換到維基百科或查看第二個視窗時，您都會受到**上下文切換懲罰**的影響。您的大腦需要幾毫秒才能重新對焦在遊戲中——但，在「極限困難模式」中，您沒有這樣的時間可以浪費。

HPO 透過以下方式讓您保持在**專注狀態**：
* **前注意處理：** 證據以特定顏色發光（🔴 紅色 = EMF 5，🔵 藍色 = 冰點）。您的大腦會立即識別這些模式，而無需閱讀文字。
* **肌肉記憶輸入：** 透過全域熱鍵（`Alt+1` 到 `Alt+7`）切換證據，無需停止移動。

---

## ⚡ 核心功能

### 1. 即時推理引擎
不用猜測。一目了然。
* **過濾器：** 當您透過熱鍵標記證據時，幽靈列表會即時過濾。
* **智能邏輯：** 在節奏計上標記「正常速度」會自動排除怨靈。標記「靈球」會排除幻影（擬魂，除非您將其標記為假靈球）。

### 2. 幽靈節奏計（BPM）
* **節奏同步：** 隨著幽靈的腳步聲節奏點擊 `F10`。
* **即時識別：** 應用程式會計算 BPM 並與您的證據列表交叉比對，即時突出顯示潛在的匹配項（例如：寒魔、魔洛伊、剎那）。

### 3. 智能聖木計時器
* **視覺提示：** 追蹤幽靈（180秒）、惡魔（60秒）、標準幽靈（90秒）的冷卻時間。
* **恐慌管理：** 精確知道何時可以安全離開您的藏身處。

### 4. 三種戰術模式
* **📖 分析模式：** 完整的日誌視圖，適合在貨車或安全點使用。
* **🧠 智能模式 [HOME]：** 緊湊的抬頭顯示介面，專為溜鬼設計。只顯示重要數據，不會遮擋您的視線。
* **⏱️ 生存模式 [PAUSE BREAK]：** 將 UI 簡化到只剩下計時器。在追逐的危急時刻實現零干擾。

## 🎮 控制與使用

**重要：** 您必須在 **無邊框視窗** 模式下執行《恐鬼症》，疊加層才會顯示。

### 全域熱鍵
| 按鍵 | 功能 |
| :--- | :--- |
| **F7** | 開始 / 重啟 十字架 計時器 (0:25) |
| **F8** | 停止 / 重置 聖木計時器 |
| **F9** | 開始 / 重啟 聖木計時器 (3:00) |
| **F10** | 點擊以測量幽靈腳步速度 (BPM) |
| **F11** | 重置 速度 / 節奏計 |
| **Alt + 1-8** | 切換證據狀態 (EMF、通靈盒、指紋等) |
| **Home** | 切換 **智能模式** (戰術 HUD) |
| **Insert** | 切換 **生存模式** (迷你計時器) |
| **滑鼠右鍵** | 關閉 |

## 🛡️ 安全與合規性
本應用程式是**外部疊加層**。
* ✅ 它**不會**注入程式碼到遊戲中。
* ✅ 它**不會**讀取遊戲記憶體。
* ✅ 它安全且不會違反反作弊政策。

## 👨‍💻 獻給開發者 (作品集備註)
此專案展示了：
* **WPF & XAML：** 複雜的 UI 綁定、透明視窗分層和動態資源字典。
* **P/Invoke (User32.dll)：** 實作全域低級鍵盤掛鉤（Global Low-Level Keyboard Hooks），用於檢測應用程式焦點之外的熱鍵。
* **MVVM 架構：** 使用 `INotifyPropertyChanged` 乾淨地分離邏輯和 UI。
* **多線程：** 高優先級的調度計時器（Dispatcher Timers）以確保在重度遊戲負載期間 UI 仍能保持響應。
